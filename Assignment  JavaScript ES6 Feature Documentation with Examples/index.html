ES6 Features Documentation with Examples

1. Closures

A closure gives you access to an outer function's scope from an inner function.


function outer() {
let naam = "Yahya"; // Outer function ka variable
function inner() { 
   console.log(naam); // Outer ke variable ko access kar raha hai
}
   return inner; // Inner function return ho raha hai
}
              
const closureFunction = outer(); // Outer ko call kiya, jo inner return karta hai
closureFunction(); // "Yahya" print hoga, kyun ki inner ke paas naam ki value preserved hai

2. Destructuring

Destructuring allows you to unpack values from arrays or properties from objects.


// Array Destructuring
const [a, b] = [1, 2];
console.log(a, b); // Output: 1, 2

// Object Destructuring
const person = { name: "Alice", age: 25 };
const { name, age } = person;
console.log(name, age); // Output: Alice, 25

3. Default Parameters

Default function parameters allow you to set initial values for function arguments.


function greet(name = "Guest") {
    console.log(`Hello, ${name}`);
}

greet(); // Output: Hello, Guest
greet("Alice"); // Output: Hello, Alice

4. Rest Parameter

The rest parameter collects all remaining elements into an array.


function sum(...nums) {
   console.log(nums); // Array of all arguments
}
              
 sum(1, 2, 3); // Output: [1, 2, 3]

5. Spread Operator

The spread operator allows you to expand elements of an array or object.


// Spread with Arrays
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5];
console.log(arr2); // Output: [1, 2, 3, 4, 5]

// Spread with Objects
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 };
console.log(obj2); // Output: { a: 1, b: 2, c: 3 }

6. Enhanced Object Literals

Enhanced object literals make it easier to write object properties and methods.


const name = "Alice";
const age = 25;

const person = {
    name, // Property shorthand
    age,
    greet() { // Method shorthand
        console.log(`Hello, I am ${this.name}`);
    }
};

person.greet(); // Output: Hello, I am Alice

7. Property Shorthand

Property shorthand allows you to shorten syntax when property names and variables match.


const name = "Bob";
const age = 30;

const user = { name, age };
console.log(user); // Output: { name: "Bob", age: 30 }

8. Method Shorthand

Method shorthand makes defining functions inside objects concise.


const car = {
    start() {
        console.log("Car started!");
    },
    stop() {
        console.log("Car stopped!");
    }
};

car.start(); // Output: Car started!

9. Computed Property Names

Computed property names allow dynamic property keys in objects.


const propName = "dynamicKey";

const obj = {
    [propName]: "Hello, World!"
};

console.log(obj.dynamicKey); // Output: Hello, World!
